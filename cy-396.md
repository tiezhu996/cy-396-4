## 项目主要需求

### 项目概述
**项目类型：纯后端服务**
搭建一个手工教程与材料商城 API 服务，为手工爱好者平台提供教程管理、材料商品管理和订单管理的后端接口。

### 功能需求

1. **教程管理**：创建、编辑、删除手工教程（标题、分类（编织/木工/陶艺/皮具/纸艺/刺绣）、难度等级、材料清单、步骤图文），支持教程审核状态管理（草稿/待审核/已发布/已下架）。
2. **教程搜索与推荐**：按分类、难度、关键词搜索教程，按热度（浏览量+收藏量）推荐热门教程。
3. **材料商品管理**：管理材料商品信息（名称、分类、价格、库存、规格参数、商品图片），支持多 SKU 管理。
4. **购物车与订单**：用户将材料加入购物车、下单购买，订单状态流转（待付款→待发货→已发货→已完成→已评价）。
5. **评价系统**：用户对教程评分评论（1-5星），对已购材料进行评价和晒单。
6. **用户关注与收藏**：关注教程作者、收藏教程和商品，个人首页展示关注动态。
7. **数据统计接口**：教程浏览排行、商品销量排行、用户活跃度统计。

### 技术要求

- 后端框架：NestJS + TypeScript
- ORM：TypeORM
- 数据库：MySQL 8.0
- 认证：JWT
- 图片存储：本地 uploads 目录
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（错误码、教程分类枚举、订单状态）统一维护在 `backend/src/constants/` 目录下；日志统一使用 NestJS 内置 Logger，日志格式配置在 `backend/src/common/logger/`；异常信息统一定义在 `backend/src/common/errors/`，通过 NestJS ExceptionFilter 统一捕获并返回标准格式响应。

## 部署要求

项目必须支持 Docker Compose 一键部署。

需要提供以下文件：
- 根目录 `docker-compose.yml`（编排后端 + 数据库）
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=craftapi`
- `backend/Dockerfile`
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: craftapi`
- 容器名带 `craftapi-` 前缀
- 数据库数据使用命名卷持久化
- 数据库配置 `healthcheck`
- 端口映射：
  - 后端：19411:3000
- 访问 API 文档（Swagger）：http://localhost:19411/api/docs
